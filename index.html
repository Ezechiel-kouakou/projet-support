Projet Hackathon BTS SIO — Version 1.0 — Octobre 2025

👋 Présentation

Dans le cadre du Hackathon BTS SIO organisé à l'Efrei, l'équipe Seyrin a constitué une entreprise fictive afin de mettre en œuvre l'ensemble des compétences du référentiel E5. Le projet couvre le déploiement d'une infrastructure complète, la création d'un site vitrine, l'intégration de GLPI et le déploiement cloud Azure.

🏢 Équipe

  • FRA — Développeur
  • Rayan — Infrastructure
  • Kylian Gabriel Henri — Support
  • Kouakou — Chef de Projet
  • Firadous — Support

🖥️ Infrastructure déployée

Trois serveurs ont été mis en ligne : Serveur GLPI → Azure | Base MySQL | Agent GLPI Linux/Windows Site vitrine → GitHub Pages | HTML/CSS/JS | Nom de domaine Backend API → Alwaysdata | Node.js / PHP | Nom de domaine

✅ Compétences couvertes par l'équipe

C1 Recensement et identification des ressources numériques

Déploiement de l'agent GLPI sur l'ensemble du parc. Attribution des rôles (Admin, Technicien, Utilisateur) pour chaque compte. Création de tous les comptes membres.

Réalisé par : Kylian, Kouakou, Firadous

C4 Continuité de service

Tests d'intégrité du serveur réalisés. Vérification des services critiques (HTTP/HTTPS, MySQL). Analyse des journaux système.

Réalisé par : FRA, Rayan, Kouakou

C5 Gestion des sauvegardes

Migration des données du serveur distant vers Azure. Exports MySQL planifiés. Snapshots des machines virtuelles.

mysqldump -u root -p seyrin_db > backup_seyrin.sql

Réalisé par : Rayan, Kouakou

C11 Référencement et visibilité en ligne

Intégration de Google Analytics sur le site vitrine. Rédaction de la politique de confidentialité conforme RGPD.

Réalisé par : FRA, Rayan, Kylian, Kouakou

C12 Évolution du site Web

Création de la première page et développement continu du site vitrine Seyrin. Intégration des API métier. Animations, responsive, modales.

Techno : HTML5 / CSS3 / JavaScript vanilla

Réalisé par : FRA, Rayan, Kylian, Kouakou, Firadous

C17 Déploiement d'un service

Déploiement du site sur Azure. Passage en HTTPS (certificat Let's Encrypt). Attribution et configuration du nom de domaine. Mise en ligne de GLPI et du backend.

certbot --apache -d seyrin.com DNS : Enregistrements A + CNAME configurés

Réalisé par : toute l'équipe

C18 Accompagnement des utilisateurs

Configuration de GLPI pour chaque membre. Vérification des accès et formation à l'outil.

Réalisé par : Rayan

🛠️ Stack technique

Frontend : HTML5 / CSS3 / JavaScript Backend : Node.js / PHP — Alwaysdata GLPI : Azure VM — MySQL — Agent Linux/Windows Hébergement : GitHub Pages + Azure Monitoring : Google Analytics SSL : Let's Encrypt (Certbot)

C11 Rédaction de la politique de confidentialité

Contexte : Dans le cadre du projet Seyrin, j'ai rédigé la politique de confidentialité du site vitrine afin d'assurer la conformité RGPD et d'informer les utilisateurs sur le traitement de leurs données.

Démarche : Identification des données collectées (nom, email, adresse IP, cookies), définition des finalités de traitement, précision des droits des utilisateurs (accès, rectification, effacement) et intégration du document dans la modale du site.

Conclusion : Cette réalisation m'a permis de comprendre les obligations légales RGPD et la notion de transparence dans le traitement des données personnelles.

C12 Amélioration du site vitrine

Contexte : Participation active à l'évolution du site vitrine de l'entreprise fictive Seyrin.

Démarche : Développement et intégration de nouvelles fonctionnalités (animations de scroll, barre de progression, modales interactives), compatibilité responsive et optimisation des performances front-end.

HTML5 / CSS3 / JavaScript vanilla

Conclusion : Renforcement des compétences en développement web et livraison d'interfaces soignées en contexte collectif.

C4 Tests d'intégrité du serveur

Contexte : Vérification de la continuité du service en testant l'état des composants critiques du serveur.

Démarche : Tests de connectivité, vérification des services (HTTP/HTTPS, base de données) et analyse des journaux système pour détecter toute anomalie.

Conclusion : Sensibilisation à la surveillance proactive des infrastructures pour garantir la continuité de service.

C12 Création des API pour les fonctionnalités

Contexte : Développement des endpoints API nécessaires aux fonctionnalités métier du site Seyrin.

Démarche : Conception et implémentation d'une API REST exposant les données de l'organisation, avec gestion des accès et documentation des routes.

POST /api/v1/contact GET /api/v1/services

Conclusion : Acquisition d'une vision concrète de l'architecture client-serveur et de la structuration d'une API.

C17 Attribution du nom de domaine

Contexte : Configuration du nom de domaine pour rendre le site vitrine accessible publiquement.

Démarche : Configuration des enregistrements DNS (A, CNAME), liaison avec l'hébergeur et vérification de la propagation DNS.

Conclusion : Compréhension du fonctionnement du DNS et des étapes de mise en ligne d'un service web.

C11 Politique de confidentialité

Contexte : Participation à la rédaction en apportant une expertise sur les aspects techniques de collecte de données (logs serveur, adresses IP).

Conclusion : Meilleure compréhension des obligations légales pour les hébergeurs et administrateurs système.

C18 Gestion des utilisateurs GLPI

Contexte : Configuration de GLPI pour s'assurer que chaque membre dispose d'un compte opérationnel avec les bons droits.

Démarche : Création des comptes, assignation des profils (Admin, Technicien, Observateur) et vérification des connexions.

sudo apt update && sudo apt install glpi-agent

Conclusion : Maîtrise de la gestion des utilisateurs dans un outil ITSM et compréhension des niveaux d'habilitation.

C4 Tests d'intégrité du serveur

Contexte : Vérification régulière de l'état du serveur Azure pour garantir la disponibilité des services.

Démarche : Scripts de monitoring, vérification des services actifs (Apache, MySQL), analyse des logs et tests de charge basiques.

Conclusion : Sensibilisation à la supervision d'infrastructure et aux indicateurs de santé système.

C12 Création des API

Contexte : Configuration du serveur backend pour le déploiement des endpoints API.

Démarche : Configuration du serveur PHP/Node.js sur Alwaysdata, déploiement des endpoints et tests de disponibilité.

Conclusion : Compétences acquises en déploiement backend et configuration de serveurs web.

C17 Passage en HTTPS

Contexte : Sécurisation du site vitrine et du serveur GLPI via un certificat SSL.

Démarche : Génération et installation d'un certificat Let's Encrypt, configuration de la redirection HTTP → HTTPS, vérification du bon fonctionnement.

certbot --apache -d seyrin.com -d www.seyrin.com

Conclusion : Compréhension approfondie de TLS/SSL et de son importance pour la sécurité des communications web.

C5 Déploiement Azure + Migration des données

Contexte : Déploiement de l'infrastructure sur Azure et migration des données depuis le serveur local vers le cloud.

Démarche : Création d'une VM Azure, configuration réseau (NSG, IP publique), migration MySQL via export/import, vérification de l'intégrité post-migration.

mysqldump -u root -p seyrin_db > backup.sql mysql -u root -p seyrin_db_azure < backup.sql

Conclusion : Expérience concrète de migration cloud et de gestion d'infrastructure Azure.
